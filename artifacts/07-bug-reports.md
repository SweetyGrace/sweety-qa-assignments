# 7. Bug Report Template & Samples

**Project:** Vendor Bill & Payment Hub
**Document Type:** Defect Template + Sample Defects
**Version:** 1.0

---

> A good defect report gets fixed. A poor one gets closed as *"Cannot Reproduce."* Every report below states exactly what was expected, what happened, and how to see it again.

## 7.1 Defect Report Template

| Field | Value |
|-------|-------|
| Defect ID | _Auto-generated by Jira_ |
| Title | _One line: Module — what happens — when_ |
| Reporter | _QA engineer name_ |
| Date Reported | _DD-MMM-YYYY_ |
| Environment | _QA / Staging / UAT · Browser · OS · Build version_ |
| Feature / Module | _e.g., Bill Submission_ |
| Requirement ID | _e.g., REQ-02_ |
| Severity | Critical / High / Medium / Low |
| Priority | P1 / P2 / P3 / P4 |
| Reproducibility | Always / Intermittent / Unable to reproduce |
| Preconditions | _System state required before reproducing_ |
| Steps to Reproduce | 1. … 2. … 3. … |
| Expected Result | _What should happen per requirement/design_ |
| Actual Result | _What actually happened_ |
| Attachments | _Screenshot / video / API response / log snippet_ |
| Assigned To | _Developer_ |
| Status | Open / In Progress / Fixed / Verified / Closed |

> **Severity vs. Priority** — Severity is the *impact* on the system; Priority is the *urgency* of the fix. They are not the same: a typo on the homepage may be low severity but high priority for a launch.

## 7.2 Sample Defects

### DEF-001 — Bill Submission: bill accepted with no file attached
| Field | Value |
|-------|-------|
| Title | Bill Submission — vendor can submit a bill without attaching any file |
| Environment | QA · Chrome 126 · Windows 11 · Build 1.4.0 |
| Feature / Requirement | Bill Submission / REQ-02 |
| Severity / Priority | High / P1 |
| Reproducibility | Always |
| Preconditions | Vendor V001 signed in; `PO-7001` open. |
| Steps | 1. Open **Submit Bill**. 2. Fill all mandatory text fields validly. 3. Do **not** attach a file. 4. Click **Submit**. |
| Expected | Validation error `Bill file attachment is required.` Form is not submitted. |
| Actual | Bill is submitted with status **Pending Review** despite no file. The record shows a blank attachment. Server does not enforce the rule. |
| Attachments | `screenshot_def001.png`, `api_response_def001.json` |

### DEF-002 — Access Control: vendor can open the AP queue by direct URL
| Field | Value |
|-------|-------|
| Title | Access Control — Vendor role reaches the AP bill queue via direct URL |
| Environment | QA · Firefox 127 · macOS 14 · Build 1.4.0 |
| Feature / Requirement | RBAC / REQ-07 |
| Severity / Priority | Critical / P1 |
| Reproducibility | Always |
| Preconditions | Vendor V001 signed in with role = Vendor. |
| Steps | 1. Sign in as V001. 2. In the address bar go to `/ap/bill-queue`. 3. Observe the rendered page. |
| Expected | Access denied — redirect to the vendor dashboard or HTTP 403. |
| Actual | The AP queue loads fully, showing **all vendors'** pending bills. Approve/Reject buttons are functional. |
| Attachments | `screenshot_def002.png` |

### DEF-003 — Bill Submission: over-billing not blocked
| Field | Value |
|-------|-------|
| Title | Bill Submission — bill exceeding remaining PO balance is accepted |
| Environment | QA · Chrome 126 · Windows 11 · Build 1.4.0 |
| Feature / Requirement | Bill Submission / REQ-02 |
| Severity / Priority | High / P2 |
| Reproducibility | Always |
| Preconditions | `PO-7002` has remaining balance ₹30,000. Vendor V001 signed in. |
| Steps | 1. Submit a bill against `PO-7002` for ₹35,000. 2. Click **Submit**. |
| Expected | Block with `Bill amount exceeds the remaining PO balance of 30,000.` |
| Actual | Bill is accepted and queued for review; no balance check is performed. |
| Attachments | `screenshot_def003.png` |

### DEF-004 — Notifications: rejection email omits the reason
| Field | Value |
|-------|-------|
| Title | Notifications — rejection email sent without the rejection reason |
| Environment | QA · Edge 125 · Windows 11 · Build 1.4.0 |
| Feature / Requirement | Notifications / REQ-05 |
| Severity / Priority | Medium / P3 |
| Reproducibility | Always |
| Preconditions | A bill is rejected by AP with reason text entered. |
